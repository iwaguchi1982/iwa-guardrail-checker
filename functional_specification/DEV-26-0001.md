iwa-guardrail-checker 仕様案 v0.1
1. 目的

生存解析や探索解析を実行する前に、入力データが

解析に耐えるか

危険な欠損やイベント不足がないか

群比較や Cox 解析に進んでよいか

を判定し、DANGER / CAUTION / OK を返す。

2. 想定ユーザー

バイオインフォエンジニア

医学・薬学系の研究者

解析前のデータ品質を確認したい人

本体アプリに入れる前段でチェックしたい人

3. 提供価値

「計算できる」ではなく「計算してよいか」を先に判断できる

欠損・イベント不足・群の偏りでの事故を減らせる

本体アプリに入れる前の前処理ゲートとして使える

単体でも価値が説明しやすい

4. v0.1 のスコープ
入力

最低限この2種類を受ける。

A. clinical table

必須候補列:

patient id

duration / follow-up time

event

group

任意列:

age

sex

stage

covariates

B. optional metadata

解析モデル種別
例: Kaplan-Meier / Cox

想定共変量数

categorical columns 指定

5. v0.1 の主機能
5.1 データ基本チェック

必須列の存在確認

患者ID重複チェック

行数 / 患者数の確認

数値列の coercion 可能性チェック

event 列の 0/1 化確認

5.2 Survival Guardrail 判定

返す指標:

total n

total events

group sizes

events per group

minimum group size

minimum group events

events per parameter

判定ロジック例:

total events < 15 → DANGER

いずれかの群で events = 0 → DANGER

min group events < 5 → CAUTION

Cox で events per parameter < 5 → DANGER

Cox で events per parameter < 10 → CAUTION

5.3 Missingness チェック

指定列ごとの missing rate

any missing row rate

severe / moderate / low の段階分類

complete-case にしたときの残存サンプル数推定

判定ロジック例:

missing > 20% → Severe

missing > 5% → Moderate

5.4 Group balance の簡易チェック

group ごとのサンプル数

stage / sex の群分布の偏り確認

age の群差を簡易確認

group size imbalance の警告

v0.1 では統計検定まで必須にしない。
まずは偏りの見える化と警告で十分。

5.5 総合判定

最終的に

OK

CAUTION

DANGER

を返し、理由を箇条書きで表示。

10. README の一行説明案

Preflight checker for survival-oriented bioinformatics datasets, with guardrails for missingness, event sufficiency, and model readiness.

日本語なら:

生存解析前のデータに対して、欠損・イベント不足・モデル適合性を事前判定するガードレールアプリ。
