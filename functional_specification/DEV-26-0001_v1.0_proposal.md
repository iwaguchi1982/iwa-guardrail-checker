# iwa-guardrail-checker Functional Specification V1.0 (Proposal)

## 1. 目的 (Core Objective)
生存解析 (KM/Cox) や探索的解析を実行する前に、入力データが統計学的な要件を満たしているか、また推定の収束を妨げる致命的な要因（欠損、イベント不足、群の偏り）がないかを自動判定し、解析の信頼性を担保する。

## 2. ターゲット (Personas)
- **バイオインフォマティシャン**: 大規模なコホートを処理する前の自動検品。
- **医学・薬学系研究者**: 統計ソフトに入れる前の「手戻り」防止。
- **データマネージャー**: 既存データベースの品質モニタリング。

## 3. 主要な更新点 (Major Enhancements in V1.0)
- **統計的検定の導入**: 単なる比率だけでなく、群間の分布差 (Chi-square / Wilcoxon) を自動計算。
- **収束整合性チェック (Cox Readiness)**: カテゴリ内のイベント数 0 や、準完全分離 (quasi-complete separation) のリスクを検知。
- **CLI / API モード**: 他のパイプラインから呼び出し可能なコマンドラインインターフェースの提供。
- **多重共線性 (Collinearity) の簡易診断**: 共変量間の相関が 0.9 を超える場合に警告。

## 4. 入力データの詳細要件
### A. Clinical Table (CSV/TSV)
- **必須列 (Mandatory)**
  - `patient_id`: ユニーク識別子
  - `time`: 生存時間（数値）
  - `event`: 生存状態（0: censored, 1: event）
  - `group`: 比較対象となる主因子
- **推奨列 (Recommended)**
  - `age`, `sex`, `stage`, `treatment` 等の共変量

### B. Config/Metadata (Optional)
- 解析モデル種別 (`KM`, `Cox`, `Multivariate`)
- 欠損値許容しきい値 (Default: 20%)
- 共変量のリスト指定

## 5. ガードレール判定ロジック
### 5.1 Survival Guardrail
| 指標 | DANGER | CAUTION | OK |
| :--- | :--- | :--- | :--- |
| **Total Events** | < 10 | < 20 | ≥ 20 |
| **Events per Group** | 0 in any group | < 5 in any group | ≥ 5 in all groups |
| **Events per Parameter (EPP)** | < 5 | < 10 | ≥ 10 |
| **Censoring Rate** | > 95% | > 85% | ≤ 85% |

### 5.2 Missingness Guardrail
- **Overall Missing Rate**: > 30% は DANGER。
- **Essential Column Missing**: `time` or `event` に欠損があれば即 DANGER。
- **Complete-case Drop**: 欠損削除後にサンプル数が元の 50% を切る場合は DANGER。

### 5.3 Categorical Covariate Readiness
- カテゴリ変数の各レベル（Stage I, II, III等）において、イベント数が 0 の場合は Cox 解析の推定が不安定になるため DANGER。

## 6. UI/UX デザイン
### 6.1 Dashboard (Summary Card)
- **Top Metrics**: `Overall Status (RED/YELLOW/GREEN)`, `Quality Score (0-100)`.
- **Quick Action**: 推奨されるアクション（「データのマージ提案」「欠損が多い列の除外提案」等）を提示。

### 6.2 Visualizations
- **Missingness Heatmap**: どの患者・どの項目に欠損が集中しているか。
- **Event Frequency bar**: 群ごとのイベント・検閲数の分布。
- **Covariate Distribution**: 群間の共変量の偏りを視覚化 (Stacked Bar / Boxplot)。

## 7. 出力 (Outputs)
- **JSON Report**: 他のシステムでパース可能な構造化データ。
- **PDF/HTML Report**: エビデンスとして保存可能なフォーマット。
- **Markdown Text**: Slack や GitHub issue に貼り付け可能なサマリー。

## 8. 技術スタック・流用計画
- **Frontend**: Streamlit (Main App との統一感)
- **Backend Analytics**: Lifelines, Pandas, Scipy
- **Component Reuse**:
  - 本体の `_assess_survival_guardrails` をリファクタリングし、`lib/guardrails.py` として独立させる。

## 9. V1.0 スコープ外
- 生存曲線 (KM curve) の描画そのもの（それは本体で行う）。
- 高度な Imputation (多重代入法)。
- 複数のコホートの同時比較。

---
## 修正のポイント (Summary of Revisions)
1. **DANGERしきい値の具体化**: 統計学的な定説 (EPP 10-15) に基づき、より厳密な判定に変更しました。
2. **Cox回帰の収束リスク検知**: 解析者が一番困る「計算が回らない」ケースを事前に防ぐ機能を追加。
3. **レポーティングの強化**: 「ただ表示する」から「結果を次に繋げる (JSON/MD)」へのシフト。

## 実装方針
**v0.1 では「判定」と「説明」に集中する。**  
解析はしない。  
図も最小限。  
機能を増やすより、**危険を止める理由が分かる**ことを優先
